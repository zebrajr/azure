Azure File Sync provides multiple ways to monitor and diagnose sync problems. Tools are provided in the sync group and are on the Windows Server computer. You can also set up alerts in Azure Monitor.

You've deployed Azure File Sync and configured both the Azure and on-premises environments. Now you need to troubleshoot a rogue file that's not syncing. You'll use the tools provided by Microsoft.

In this exercise, you'll first check that file sync is working correctly. Then you'll generate a file with invalid characters. You'll see how to diagnose sync problems and resolve them. The following video walks you through the process. Complete the steps in this exercise to continue following along in your Azure subscription.

Find your connection details

  1. Sign in to the Azure portal.

  2. Go to the storage account. Select All resources > extendcadfiles.

  3. Under Settings, select Access keys.

  4. Under key1, copy the Key value. You'll use this key to set up the mapped drive.

  1. If you closed your Remote Desktop connection, use these steps to connect to the VM:
    1. Sign in to the Azure portal.
    2. Go to Virtual machines > FileServerLocal.
    3. Select Connect > Download RDP File, and then open the RDP file.
    4. In the Windows Security window, select More choices > Use a different account.
    5. Enter the user name learnadmin and the password you used when you created the VM.
  2. On the toolbar, select File Explorer, and then select This PC.
  3. In the menu at the top of the window, select Computer > Map network drive.
  4. In the Folder box, enter \\extendcadfiles.file.core.windows.net\cadfileshare.
  5. Select Connect using different credentials.
  6. Select Finish.
  7. For the user name, enter AZURE\extendcadfiles.
  8. For the password, paste in the key you copied in the previous section.
  9. Select OK.

Check the file sync

  1. Open a new File Explorer window.

  2. Put the two File Explorer windows side by side, with D:\CADFolder on the left and the mapped cadfileshare on the right:

  3. In the left window, below CADFolder, right-click and select New > Text Document.

  4. Rename the file to test.

  5. Confirm that the file replicates to the Azure file share in the right window.

Troubleshoot file synchronization

You can troubleshoot on your on-premises server or in the Azure portal. Syncing to the portal can take some time, so start troubleshooting locally.

View the file sync issue on Windows Server

  1. Right-click Start and select Windows PowerShell (Admin).

  2. In the console, move to the File Sync folder:

    cd 'C:\Program Files\Azure\StorageSyncAgent\'

  3. Run the error report script:

    .\FileSyncErrorsReport.ps1

  4. View the output of the command. The ErrorDescription field describes what's wrong. The ItemPath field gives the location of the problematic file.

  5. Verify that the error is 0x8007007b and that it refers to the invalid file you generated.

View the file sync problem in the Azure portal

  1. On your local computer, return to the Azure portal.

  2. Go to the sync group. Select All resources > CADSyncService > CADFiles.

  3. Under server endpoints, select FileServerLocal.

  4. Next to Transient, select 1 files. It might take a few minutes for the file to show up.

  5. Under Remediation, Azure provides advice on how to resolve the problem.

Because the file sync problems just started, they're listed as transient. If the file remained in the folder with the invalid characters for more than 24 hours, the status would change to persistent.

Resolve the file sync

  1. Return to the Remote Desktop connection.

  2. Remove the invalid characters from the file name. In the left File Explorer, rename the invalid╘«nameÉ.txt file to valid-name.

  3. The file should now sync. Confirm the sync works by running this command again:

    .\FileSyncErrorsReport.ps1
