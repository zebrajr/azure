Secure guest access - Learn

lizap
4-5 minutes
  • 5 minutes

Microsoft Teams allows you to collaborate with people both inside and outside your organization. The people outside your organization must have guest access. Many organizations find this guest access vital, but you must take care to ensure that it doesn't introduce vulnerabilities into your system.

How to secure guest access

There are seven steps. You'll find a link to a troubleshooting guide in the Learn more section.

  1. Enable Guest access in Teams.
  2. Configure Azure AD to allow guests in Teams.
  3. Configure Microsoft 365 groups.
  4. Configure Sharing in Microsoft 365.
  5. Verify Sharing settings in SharePoint.
  6. Set up Guest permissions in Teams.
  7. Turn on Anonymous users can join a meeting if necessary.

Let's examine these steps in more detail.

Enable guest access in Teams

To allow or disallow guest access for the whole organization, you must be a Teams service administrator.

  1. Sign into the Teams admin center.
  2. Select Org-Wide settings > Guest access.
  3. Set the Allow guest access in Microsoft Teams switch to On.
  4. On the same page, you enable or disable guests, or turn off Calling, Meeting, and Messaging settings for guests.
  5. Select Save. Each Team can then choose whether to allow guess access.

Configure Azure AD

Before you collaborate with guests in Teams, you must configure your Azure AD settings.

  1. Sign into the Azure portal as a tenant administrator.
  2. Select Azure Active Directory > Users > User settings.
  3. Under External users, select Manage external collaboration settings. You can also access the same settings from Manage > Organizational relationships > Settings.
  4. On the External collaboration settings page, choose the policies you want to enable.

Configure Microsoft 365 Groups

If you want guest users to access group content, you must specifically enable them.

  1. In the Microsoft 365 admin center, go to Settings > Org Settings.
  2. Select Services, and then select Microsoft 365 Groups.
  3. Select the check box Let group members outside the organization access group content to allow guests to access group content.
  4. Select the check box Let group owners add people outside the organization to groups to allow team owners to add new guests. This setting must be turned on to support guest access.

Configure sharing in Microsoft 365

You can enable ordinary users to add guests:

  1. In the Microsoft 365 admin center, go to Settings > Org Settings.
  2. Select Security & privacy > Sharing.
  3. Select the check box Let users add new guests to this organization > Save changes.

Because Teams uses SharePoint sites for storage, verify the guest settings for those sites:

  1. In the Microsoft 365 admin center, under Admin centers, select SharePoint.
  2. Under Sites, select Active sites.
  3. Select the site, select Sharing, and then select either Anyone or New and existing guests.

Set guest permissions in Teams

In the Teams application, at the individual team level, configure guest permissions that control whether guests can create, update, or delete channels. Teams administrators and team owners can configure these settings.

Anonymous users can join a meeting

If you want guests to join meetings, you must turn on the Anonymous users can join a meeting setting in Microsoft Teams.

  1. In the Teams admin center, go to Meetings > Meeting settings.
  2. Under Participants, select Anonymous users can join a meeting.
